Introduction to Controllers

  • Controllers are JavaScript functions
  • They have to be declared in your module to be accessible
  • They are used to:
    • Set up the initial state of a $scope object.
    • Add behavior to the $scope object
    • They don't have to contain any business logic, but do the link with your services, which contain business logic
  • Remark: use them as less as possible because they will be removed with angular 2
angular.module('sample', []) .controller('ClassicController', function($scope) { $scope.welcomeMsg = 'Hello World'; $scope.onButtonClicked = function() { console.log('button clicked'); }; }) .service('myService', function(){ this.welcomeMsg= 'Hello world'; this.onButtonClicked= function(){ console.log('button clicked'); }; }) .controller('BetterController', ['$scope','myService', function($scope, myService){ $scope.service= myService; }]); <div ng-app="sample" > <div ng-controller="ClassicController"> <input type="text" ng-click="onButtonClicked()"/> {{welcomeMsg}} </div> <div ng-controller="BetterController"> <input type="text" ng-click="service.onButtonClicked()"/> {{service.welcomeMsg}} </div> </div>
