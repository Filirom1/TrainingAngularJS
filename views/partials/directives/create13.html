The require attribute

  • It's possible to have dependencies between directives As an example, in this support, the editor is a directive, and a tab is a directive inside editor
  • In such a case, the require attribute specifies that a directive requires an other one
  • This declaration implies that the controller of the required directive will be injected in the one which requires it (as the fourth argument of the linking function)
  • If no controller is found, an error is raised. You can use ? prefix to make the dependency optional.
  • Use ^ prefix to look for the controller on parent elements as well.
application.directive('ngSwitchDefault', function() { return { require: [ '^ngSwitch' ], link: function (scope, element, attrs, controllers) { // controllers[0].callMethod(...); } }; }); application.directive('input', function() { return { require: '?ngModel', link: function (scope, element, attrs, controllers) { } }; });
