The transclude attribute

  • In previous samples, we have seen that the elements contents were never visible, and we always replaced by the template content
  • transclude allows you to retrieve the element content Ok, but where the element content will be added if you also have a template? The answer is, where you want. You have to use the ng-transclude directive into your template to specify where the element content will be inserted.
  • You can also do your transclude "dynamically" Below are 2 directives examples:
    • The first one corresponds to the "basic" usage, with transclude:true and the ng-transclude directive
    • The second is not common, and not easy, and less important to understand, but show you how you can dynamise the transclusion behavior, by modifying the dom yoursel (choose where to do the transcluded content, and even modify it). Be sure that it's an advanced concept, not for first usage.
angular.module('app',[]) .directive('classicTransclude', function () { return { restrict: 'A', transclude: true, template: '<b>Directive template content with transclude: ' + '<div ng-transclude>here</div> </b>' }; }) .directive('advancedTransclude', function () { return { restrict: 'A', transclude: true, template: '<b>Directive template content ' + '<div id="insertHere">here</div> </b>', link: function($scope, $element, $attrs, $ctrls, $transclude){ $transclude(function(clone, scope){ var newContent= angular.element('<p>' + clone.text() + ' modified </p>'); $element.append(newContent); }); } }; }); <div ng-app="app" > <h1> Classic usage </h1> <span classic-transclude> <i>Initial transcluded content</i> </span> <h1> Advanced usage </h1> <span advanced-transclude> <i>Initial transcluded content</i> </span> </div>
