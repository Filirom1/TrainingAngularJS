Global architecture in terms of modules and dependencies

  • Each new application is a module and has to be declared
  • But a module is not necessarilly an application, it can be a library which provides services,controllers, directives and filters
  • If you can use libraries modules into an application module, it means that you can specify the libraries modules on which the application module depends
  • You have to make your applications flexible by decomposing them into modules, themselves composed of services, directives, filters and controllers reusable anywhere
// Here is your library module, which doesn't // depend on anything (but it could) angular.module('myLib',[]) // On which you can declare components such as directives, services... .service('serv1', ....).service('serv2',....) .directive('d1', ...).directive('d2', ...); //Now your new application module, which depends on several other modules angular.module('myApp', ['myLib', 'otherModule', ...]) // can use the services declared in the modules in dependency .service('yourService', ['serv1', function(serv1){ //Your service content, which can use the 'serv1' service ... }]);
