Expressions in AngularJS

  • Keep your expressions simple!
    • Less code as possible in the view.
    • No unit tests for the code in the views.
    • Prefer using controller functions.
    • Harder to debug!
//Expression only in the view... {{isUserLogged == true && ("Hello, " + currentUser.gender + " " + currentUser.firstName + " " + currentUser.lastName) || "You need to log in."}} // ...Or using controller functions.
