Use your controllers correctly!

  • The controller only goal is to contain the minimal logic needed to render the associated view
  • Do not use them for:
    • Instantiate/manage the life-cycle of other components (such as services).
    • Input formatting (Use form controls instead).
    • Output filtering (Use Angular filters instead).
    • To run stateless or stateful code shared across controllers (Use Angular services instead).
    • DOM manipulation.
