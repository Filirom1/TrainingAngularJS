Creating Angular Services

  • Services are singletons
  • They offer a way for separating logic from your controllers and can be used almost everywhere in your application (DI)
  • Angular provides several ways to create your services, from the most easy to the more complex
  • Warning: Prefix your services names by something specific, because 2 services with the same name are not differentiables event if they are in different modules
  • Each service has a provider, either created manually, either created by Angular. Create your own provider is only mandatory when you want your service to be configurable at application bootstrap
